> 이 문서의 역할: exec-plans 폴더 사용법 안내. 진행 중인 작업 계획을 에이전트와 공유하는 시스템.

# 작업 계획서 (Exec Plans)

## 폴더 구조

```
exec-plans/
└── active/     # 현재 진행 중인 작업 계획
```

## 사용법

1. `active/` 폴더에 작업 계획서를 만든다
2. 에이전트는 세션 시작 시 `active/` 폴더를 확인하여 진행 중인 작업 맥락을 파악한다
3. PR 머지 전 `active/`의 해당 파일을 삭제하고 push한 뒤 머지한다

## 작업 계획서 템플릿

파일명: `YYYY-MM-DD-작업명.md` (예: `2026-04-10-notification-refactor.md`)

```markdown
# [작업명]

## 목표
이 작업으로 달성하고자 하는 것

## 현재 상태
지금까지 완료된 것, 중단된 이유

## 남은 작업
- [ ] 해야 할 일 1
- [ ] 해야 할 일 2
- [x] 완료된 일

## 관련 파일
- `src/screens/...`
- `src/hooks/...`
- `src/components/...`

## 완료 기준
이 작업이 끝났다고 볼 수 있는 조건

## 메모
구현하면서 결정한 사항, 주의사항
```

## 팁

- 한 번에 하나의 작업 계획서만 `active/`에 두는 것을 권장
- 작업이 중단되었을 때 "현재 상태"를 업데이트해두면 다음 세션에서 빠르게 재개 가능
- 백엔드 팀과 협의가 필요한 사항은 메모에 별도 표시
