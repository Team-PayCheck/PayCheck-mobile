# 프로덕트 명세서

## 용도

기능 명세서, 유저 스토리, 화면 기획서를 이 폴더에 저장합니다. Notion이나 Figma에 있는 내용을 에이전트가 접근할 수 있도록 이 폴더에 옮겨두면 됩니다.

## 파일명 규칙

`{기능영역}-{기능명}.md`

예시:
- `worker-salary-statement.md` — 근로자 급여명세서 기능 명세
- `employer-remittance.md` — 고용주 송금관리 기능 명세
- `common-notification.md` — 공통 알림 기능 명세

## 명세서 권장 구조

```markdown
# [기능명]

## 개요
이 기능이 해결하는 문제

## 사용자 스토리
- 근로자로서, 나는 ... 을 원한다
- 고용주로서, 나는 ... 을 원한다

## 화면 목록
- 화면 1: 설명
- 화면 2: 설명

## 핵심 플로우
1. 사용자가 ...
2. 시스템이 ...

## 엣지 케이스
- 근무지가 없을 때
- 데이터가 없을 때

## 백엔드 의존성
- API: ...
- 백엔드 팀 협의 필요 사항: ...
```
